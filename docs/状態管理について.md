## 状態管理の設計方針

**`platform_requests` が正源、`saas_requests` はクライアント向けビュー**

```
ブラウザ          SaaS Backend          Platform          DB
  │                    │                    │               │
  │ POST /api/files     │                    │               │
  │───────────────────▶│                    │               │
  │                    │ INSERT pending ─────────────────────▶ saas.saas_requests
  │                    │ publish NATS event │               │
  │ 202 {requestId}    │                    │               │
  │◀───────────────────│                    │               │
  │                    │                    │               │
  │                    │     NATS event ───▶│               │
  │                    │                    │ UPSERT ────────▶ platform.platform_requests
  │                    │                    │               │  (pending → completed)
  │                    │ PATCH /api/requests/:id/status     │
  │                    │◀───────────────────│               │
  │                    │ UPDATE status ───────────────────────▶ saas.saas_requests
  │                    │                    │               │  (pending → completed)
  │ GET /api/files/:id │                    │               │
  │───────────────────▶│                    │               │
  │                    │ SELECT ─────────────────────────────▶ saas.saas_requests
  │ {status:completed} │                    │               │
  │◀───────────────────│                    │               │
```

**変更ファイル一覧**

| ファイル | 変更内容 |
|---------|---------|
| requests.ts | 新規: `PATCH /api/requests/:requestId/status` エンドポイント |
| index.ts | `/api/requests` ルーターを登録 |
| config.ts | `saasBackendUrl` フィールドを追加 |
| persistenceActivity.ts | `platform_requests` upsert 後に SaaS Backend へコールバック (ベストエフォート) |
| index.ts | `persistRequestActivity` の型に `result?` 引数を追加 |
| platformWorkflow.ts | 全 `persistRequestActivity` 呼び出しに `result` メッセージを渡すよう更新 |
| .env | `SAAS_BACKEND_URL=http://localhost:3001` を追加 |
